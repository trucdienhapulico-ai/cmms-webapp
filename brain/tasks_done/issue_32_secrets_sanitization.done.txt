ISSUE_32: SECURITY HARDENING - SECRET MANAGEMENT & DATA SANITIZATION

MÔ TẢ:
1. Gỡ bỏ các thông tin nhạy cảm (SSH password, Cloudflare token) khỏi mã nguồn.
2. Triển khai biến môi trường (process.env) trong deploy.js và các script liên quan.
3. Cập nhật .gitignore để chặn tracking data/*.json và các file backup.
4. Xóa (git rm --cached) các file dữ liệu/backup đã lỡ commit lên GitHub.

YÊU CẦU:
- Không để lộ secret trong code.
- Phải có file .env.example mẫu.
- GitHub không còn chứa file .json nhạy cảm.
